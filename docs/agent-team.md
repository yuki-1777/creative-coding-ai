# エージェントチームガイドライン

Claude Code の Agent Teams 機能を使ってスケッチをバッチ作成するときのルール。

**前提**: `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` が有効になっていること。

---

## チーム構成

```
Lead（あなたのメインセッション）
  ├─ Teammate A → スケッチ NN 担当
  ├─ Teammate B → スケッチ NN+1 担当
  ├─ Teammate C → スケッチ NN+2 担当
  └─ ...
```

- **Lead**: チームメートの生成・タスク割り当て・works.json の最終確認
- **Teammate（1名 = 1スケッチ）**: コンセプト決定 → 技術選定 → 実装 → サムネイル取得

各チームメートは独立したセッションで動く。`CLAUDE.md` は自動でロードされる。

---

## 各チームメートがやること

1. `docs/creative-guideline.md` を読む（**最優先**。CLAUDE.md から参照されているが、必ず自分で読む）
2. 担当番号のコンセプトを決める
3. 技術を選定する（下記「技術選定の基準」を参照）
4. `sketches/NN_name/index.html` を実装する
5. `works.json` への追記内容を Lead に報告する（直接書かない）
6. サムネイルを取得する（`/capture-thumb` スキルを使う）

> **works.json は Lead がまとめて書く。** 複数チームメートが同時に書くと競合するため。

---

## 技術選定の基準

技術はコンセプトに対して「一番面白く見える選択」をする。
事前に決めない。担当チームメートが自分で判断する。

| 技術 | 向いている表現 |
|------|--------------|
| **p5.js** | インタラクティブな2D・パーティクル・有機的成長・手描き的な線 |
| **three.js** | 3D空間・立体物・カメラが回るもの・ライティングが必要なもの |
| **glsl** | 全面シェーダー・数学的パターン・フラクタル・ノイズ場 |
| **canvas** | 物理シミュレーション・グリッド・セルオートマトン・タイルパターン |

複数の技術を使う場合は `tech` を配列で記述する：

```json
{ "tech": ["p5.js", "three.js"] }
```

---

## サムネイル取得

実装完了後、`/capture-thumb` スキルを使ってサムネイルを取得する。

- three.js・GLSL は `preserveDrawingBuffer: true` が実装されているか先に確認する

---

## 起動テンプレート

以下をそのままコピーしてメインセッションに貼り付ける。
`[NN]〜[MM]` に番号を入れるだけ。

```
スケッチ [NN]〜[MM] 番を作成してください。

## チーム構成
1スケッチ = 1チームメートで並列実行。

## 各チームメートへの指示
1. docs/creative-guideline.md を必ず読む
2. コンセプトを自分で決める（テーマは自由、アンチパターンに注意）
3. docs/agent-team.md の技術選定基準を参照して技術を選ぶ
4. sketches/NN_name/index.html を実装する
5. works.json への追記内容（1オブジェクト）を Lead に報告する
6. /capture-thumb スキルでサムネイルを取得する

## Lead への指示
全チームメートの完了後、報告された内容を works.json にまとめて追記する。
番号の重複・欠番がないか確認すること。
```

---

## 注意事項

- **works.json は Lead がまとめて書く**（競合防止）
- 開始前に `works.json` の末尾番号を確認して、番号を正しく割り当てる
- チームメートは独立したコンテキストで動くため、Lead との会話履歴は引き継がれない
- トークン消費はチームメート数に比例して増える（3〜5名が目安）
