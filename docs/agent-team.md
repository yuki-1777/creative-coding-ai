# エージェントチームガイドライン

スケッチをバッチで作成するときのエージェントチーム運用ルール。

---

## チーム構成

```
オーケストレーター（1名）
  └─ 実装エージェント × N名（並列）
       └─ 各自が1スケッチを担当
```

### オーケストレーターの役割

- 次に作る番号の範囲を確認する（`works.json` の末尾を読む）
- 各実装エージェントに番号・大まかなテーマの方向性を割り当てる
- 全エージェントの完了後、`works.json` への追記が重複・欠番なく行われたか確認する

### 実装エージェントの役割（1名 = 1スケッチ）

各エージェントが担当する作業の全量：

1. `docs/creative-guideline.md` を読む（**必須・最優先**）
2. `docs/requirements.md` を読む
3. `CLAUDE.md` を読む
4. コンセプトを決める（テーマから何を表現するか）
5. 技術を選定する（下記「技術選定の基準」を参照）
6. `sketches/NN_name/index.html` を実装する
7. `works.json` に1エントリ追記する
8. サムネイルを取得する（`/capture-thumb` スキルを使う）

---

## 技術選定の基準

技術はコンセプトに対して「一番面白く見える選択」をする。
事前に割り当てない。担当エージェントが自分で決める。

| 技術 | 向いている表現 |
|------|--------------|
| **p5.js** | インタラクティブな2D・パーティクル・有機的成長・手描き的な線 |
| **three.js** | 3D空間・立体物・カメラが回るもの・ライティングが必要なもの |
| **glsl** | 全面シェーダー・数学的パターン・フラクタル・ノイズ場・リアルタイム演算 |
| **canvas** | 物理シミュレーション・グリッド・セルオートマトン・タイルパターン |

複数の技術を組み合わせる場合は `tech` を配列にする：

```json
{ "tech": ["p5.js", "three.js"] }
```

単一技術の場合も配列が望ましいが、文字列でも動作する：

```json
{ "tech": "p5.js" }
```

---

## サムネイル取得

実装完了後、必ず `/capture-thumb` スキルを使ってサムネイルを取得する。

- スキルが自動的にキャプチャして `img/thumb_NN.png` に保存する
- three.js・GLSL の場合は `preserveDrawingBuffer: true` が実装されているか先に確認する

```
## ルール
- docs/creative-guideline.md を全員が読むこと（最優先）

- 実装後は /capture-thumb スキルでサムネイルを取得する
- works.json への追記は番号の重複・欠番がないように
```

---

## 注意事項

- **works.json の末尾番号を必ず確認してから開始する**（番号衝突防止）
- 各エージェントは他エージェントの `works.json` 追記と競合しないよう、追記のタイミングをずらすか、オーケストレーターがまとめて登録する
- アンチパターンは `docs/creative-guideline.md` の末尾を参照
