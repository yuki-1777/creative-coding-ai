# エージェントチームガイドライン

サブエージェントを使ってスケッチをバッチ作成するときのルール。

---

## チーム構成

```
オーケストレーター（親）
  ├─ サブエージェント A → スケッチ NN 担当
  ├─ サブエージェント B → スケッチ NN+1 担当
  └─ ...（並列実行）
```

- **オーケストレーター**: 番号割り当て・サブエージェント起動・works.json への最終書き込み
- **サブエージェント（1名 = 1スケッチ）**: ドキュメント読む → コンセプト → 技術選定 → 実装 → サムネイル → 結果を親に返す

> **works.json はオーケストレーターがまとめて書く。** サブエージェントは追記内容を返すだけ。

---

## 技術選定の基準

コンセプトに対して「一番面白く見える選択」をする。事前に決めない。

| 技術 | 向いている表現 |
|------|--------------|
| **p5.js** | インタラクティブな2D・パーティクル・有機的成長 |
| **three.js** | 3D空間・立体物・カメラが回るもの |
| **glsl** | 全面シェーダー・数学的パターン・フラクタル・ノイズ場 |
| **canvas** | 物理シミュレーション・グリッド・セルオートマトン |

複数技術を使う場合は配列で記述する: `"tech": ["p5.js", "three.js"]`

---

## サムネイル取得

実装完了後、`/capture-thumb` スキルを使う。
three.js・GLSL は `preserveDrawingBuffer: true` が実装されているか先に確認する。

---

## 起動テンプレート

### オーケストレーターへの指示（ユーザーが送る）

```
スケッチ [NN]〜[MM] 番を作成してください。
docs/agent-team.md に従ってサブエージェントを並列起動してください。
```

### サブエージェントへの指示（オーケストレーターが送る）

```
以下を読んでからスケッチ [NN] を作成してください：
- docs/creative-guideline.md（必読）
- docs/requirements.md
- CLAUDE.md

コンセプト・技術は自分で決める。
完了後、works.json に追記する1オブジェクトを返すこと（直接書かない）。
/capture-thumb スキルでサムネイルを取得すること。
```

---

## 注意事項

- 開始前に `works.json` の末尾番号を確認して番号を割り当てる（重複・欠番防止）
- サブエージェントは独立したコンテキストで動くため、親の会話履歴は引き継がれない
