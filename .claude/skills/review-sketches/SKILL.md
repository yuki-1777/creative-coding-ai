# Review Sketches

ユーザーがギャラリーのFBパネルからコピーしたJSONを受け取り、各スケッチのコードを読んで修正方針を出力するスキル。

## ワークフロー

ユーザーは以下の手順でFBをチャットに送る：

1. ギャラリー（`index.html`）をデバッグモードで開く
2. 右下の「FB」ボタンをクリック
3. オーバーレイの「Copy JSON」ボタンを押す
4. チャットにペースト

## 手順

### 1. JSONを受け取る

ユーザーがペーストしたJSONを読む。形式：

```json
{
  "05": "パーティクル多すぎ、重い",
  "09": "色が単調、HSBで揺らしたい"
}
```

FBが空（`{}`）の場合はその旨を伝えて終了。

### 2. works.json でスケッチのパスを解決する

`works.json` を Read ツールで読み、FBがある番号に対応するエントリの `path` フィールドを特定する。

```json
{ "num": "05", "path": "sketches/05_particle_storm/", ... }
```

### 3. 各スケッチのコードを読んで修正方針を出力

FBがある各スケッチについて:
- `works.json` で解決したパスの `index.html` を Read ツールで読む
- FBコメントと照合して修正方針を考える

### 4. 出力フォーマット

```
## NN — タイトル
> FBコメント（原文）

修正方針:
- 具体的な変更点1
- 具体的な変更点2
```

## 注意事項

- 修正は行わない（方針を出力するだけ）。実装は別途ユーザーの指示を待つ
- FBが0件の場合はその旨を伝えて終了
